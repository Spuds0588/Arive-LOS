Fix Arive Native E-Sign Not Showing! #Arive #LoanOfficer #MortgageTech

Author: Corey Burns | Updated: October 2025

Overview

Loan officers using the Arive platform may encounter a common issue where attempting to create a borrower e-sign request unexpectedly redirects them to DocuSign, bypassing the integrated Arive Native E-Sign feature. This interruption disrupts the workflow and can cause confusion. Fortunately, the problem is usually the result of a simple configuration setting that can be easily corrected. This guide walks through the necessary steps to re-enable Arive Native E-Sign as the default tool, allowing users to keep the entire signature process within the Arive ecosystem for a more efficient and streamlined experience.

Key Takeaways

  • Arive can sometimes default to an external tool like DocuSign for e-signatures instead of its built-in feature.
  • The issue is typically a settings configuration problem, not a system-wide bug.
  • Re-enabling Arive Native E-Sign is a quick fix that can be done directly within the platform's settings.
  • Using the native e-sign functionality streamlines the entire loan origination workflow, improving productivity.

    Frequently Asked Questions

    Q: Why is Arive sending my e-sign requests to DocuSign instead of using its native feature?
    A: If Arive is redirecting to DocuSign, it's likely that Arive Native E-Sign has been disabled or is not configured as the default e-signature provider in your settings. This can sometimes occur after a system update or during initial account setup.

    Q: How do I fix the Arive Native E-Sign issue?
    A: You can resolve this by navigating to your account or company settings within Arive, finding the e-signature or integrations section, and explicitly selecting 'Arive Native E-Sign' as your default provider. This ensures all future requests stay within the platform.
